# PRD-097: Semantic List Markup for Thread List

## Status
Done

## Priority
P3-Low

## Category
Accessibility

## Source
GAP-FINDER A3

---

## Problem Statement
The thread list renders as a series of plain `<div>` elements without semantic list roles. Screen readers cannot announce the list structure (e.g., "list with 50 items") or allow users to navigate by list item. This makes it difficult for screen reader users to understand the context and size of their email list.

## Current Behavior
In `thread-list.tsx`, the outer container (line 261) is `<div className="divide-y">` with no `role` attribute. Each `ThreadRow` component (line 76) renders as a `<div>` with `role="button"` and `tabIndex={0}`. The structure is flat divs with no list semantics. Screen readers announce each thread as an individual button with no sense of list position or total count.

## Expected Behavior
- The thread list container should have `role="list"` (or use a `<ul>` element)
- Each thread row should be wrapped in an element with `role="listitem"` (or `<li>`)
- Screen readers should announce "list, 50 items" when entering the thread list
- Users should be able to navigate between items using screen reader list navigation commands

## Files to Modify
- `frontend/components/thread-list.tsx:261` -- Change outer `<div>` to `<div role="list">` or `<ul>`
- `frontend/components/thread-list.tsx:262-278` -- Wrap each `ThreadRow` in a `<div role="listitem">` or `<li>`
- `frontend/components/thread-list.tsx:76-239` -- Keep the inner `<div role="button">` inside the listitem wrapper
- `frontend/components/thread-list.tsx:261` -- Add `aria-label="Email threads"` to the list container

## Implementation Notes
- Using `role="list"` and `role="listitem"` on `<div>` elements avoids styling issues that `<ul>`/`<li>` can introduce with grid layouts
- The `ThreadRow` itself should remain `role="button"` (or could become `role="row"` if implementing a grid pattern, but list is simpler)
- Nesting: `<div role="list"> > <div role="listitem"> > <div role="button" ...ThreadRow...>`
- Consider adding `aria-label` to each listitem with the thread subject for quick navigation: `aria-label="{subject} from {sender}"`
