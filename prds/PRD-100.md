# PRD-100: Color Contrast WCAG Compliance

## Status
Done

## Priority
P3-Low

## Category
Accessibility

## Source

## Problem Statement
The app uses `text-muted-foreground` extensively (178 occurrences across 33 files) with additional opacity modifiers like `/60`, `/70`, and `/30` that further reduce contrast. Many of these instances likely fail the WCAG AA 4.5:1 contrast ratio requirement for normal text. The muted foreground color values in light mode (`215.4 16.3% 46.9%` = approximately #6b7280) and dark mode (`215 16% 57%` = approximately #8b95a5) already sit near the contrast boundary before opacity reduction.

## Current Behavior
In `frontend/app/globals.css:18`, the light mode `--muted-foreground` is `215.4 16.3% 46.9%` (a medium gray). In dark mode (line 41), it is `215 16% 57%`. These are used via `text-muted-foreground` in Tailwind. Additionally, 9 instances use opacity modifiers like `text-muted-foreground/60` (in `thread-list.tsx`, `thread-view.tsx`, `settings-modal.tsx`) which reduce the opacity to 60%, making the effective contrast much lower. Examples include:
- Thread snippets: `text-muted-foreground/70` in `thread-list.tsx:175`
- Thread timestamps: `text-muted-foreground` in `thread-list.tsx:184`
- Collapsed email snippets: `text-muted-foreground/70` in `thread-view.tsx:395`
- Star button unfocused state: `text-muted-foreground/30` in `thread-list.tsx:125`

## Expected Behavior
- All text elements should meet WCAG AA 4.5:1 contrast ratio for normal text
- Large text (18px+ or 14px+ bold) needs only 3:1
- Opacity modifiers on text should not reduce contrast below these thresholds
- The base `--muted-foreground` values should be adjusted to ensure sufficient contrast

## Files to Modify
- `frontend/app/globals.css:18` -- Adjust `--muted-foreground` in light mode for better contrast (e.g., from `46.9%` lightness to `40%`)
- `frontend/app/globals.css:41` -- Adjust `--muted-foreground` in dark mode for better contrast (e.g., from `57%` lightness to `65%`)
- `frontend/components/thread-list.tsx:125,147,175` -- Replace `/30`, `/60`, `/70` opacity modifiers with full-strength `text-muted-foreground` or a higher-contrast variant
- `frontend/components/thread-view.tsx:395,398` -- Replace `/70` opacity on collapsed email text
- `frontend/components/settings-modal.tsx` -- Audit all `/60` and `/70` uses

## Impact Analysis

### What This Affects
- Visual appearance of secondary text throughout the app
- Readability for users with low vision or in bright environments
- WCAG 2.1 Level AA compliance (Success Criterion 1.4.3: Contrast Minimum)

### What Affects This
- Theme system -- changes to CSS variables affect both light and dark modes
- Any component using `text-muted-foreground` will be impacted by the base color change
- Design aesthetic -- increasing contrast may make the UI feel less "subtle"

### Related PRDs

- PRD-069 (RTL Text) -- text rendering improvements

## Implementation Notes
- Run a contrast checker (e.g., WebAIM Contrast Checker) on the current `--muted-foreground` values against `--background`:
  - Light: `#6b7280` on `#ffffff` = ~4.6:1 (borderline pass at AA)
  - Light with `/70` opacity: effective ~3.2:1 (FAIL)
  - Dark: `#8b95a5` on `#191b1f` = ~4.8:1 (borderline pass)
  - Dark with `/70` opacity: effective ~3.4:1 (FAIL)
- Strategy: Increase base `--muted-foreground` contrast slightly and eliminate opacity modifiers below `/80`
- Replace opacity modifiers with a new CSS variable `--muted-foreground-subtle` that still meets 4.5:1
- Consider auditing with the `axe-core` or `Lighthouse` contrast checker for a full list of violations
- The star button at `/30` opacity is decorative/interactive (not text), so 3:1 may be acceptable as a UI component
