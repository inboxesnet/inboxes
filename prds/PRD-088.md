# PRD-088: Cancellation UI

## Status
Done

## Priority
P1-High

## Category
Billing & Stripe

## Source
GAP-FINDER B8

---

## Problem Statement
Deleting an organization immediately cancels the Stripe subscription with no confirmation dialog that explains the consequences. There is no separate "cancel subscription" flow -- the only way to cancel billing is to delete the entire org. Users who want to downgrade to free (cancel subscription) must delete their org, losing all their data. The current `confirm()` dialog is a basic browser prompt that doesn't mention Stripe cancellation or the 7-day grace period.

## Current Behavior
In `frontend/components/settings-modal.tsx:575-585`:
```typescript
async function handleDeleteOrg() {
    if (!confirm("Are you sure you want to delete this organization? This action cannot be undone. All users will be disabled.")) return;
    // ...
    await api.delete("/api/orgs");
    window.location.href = "/login";
}
```

The confirmation uses `window.confirm()` with a generic message. It does NOT mention:
- That the Stripe subscription will be cancelled immediately (not at period end)
- That there is a 7-day grace period before access is lost
- That all queued emails will be cancelled
- That domain data, threads, and emails will be soft-deleted

On the backend (`backend/internal/handler/orgs.go:134-148`), the subscription is cancelled immediately via `subscription.Cancel(*stripeSubID, nil)`, not at the period end. This means users lose the remaining time they already paid for.

## Expected Behavior
1. Org deletion should use a proper confirmation dialog (not `window.confirm`) that clearly lists consequences:
   - "Your Stripe subscription will be cancelled immediately"
   - "You have 7 days to re-subscribe before losing access"
   - "All pending emails will be cancelled"
   - "All users will be disabled"
2. Consider adding a separate "Cancel Subscription" option in the billing tab that:
   - Cancels at period end (not immediately): `subscription.Cancel(subID, &stripe.SubscriptionCancelParams{InvoiceNow: stripe.Bool(false)})`
   - Keeps the org active until the period ends
   - Redirects to the Stripe billing portal for self-service cancellation
3. The Stripe billing portal (already available via `POST /api/billing/portal`) supports cancellation with prorated refunds -- this should be the primary cancellation path.

## Files to Modify

## Impact Analysis

### What This Affects
- User experience during org deletion
- Stripe subscription lifecycle (immediate vs end-of-period cancellation)
- Data loss prevention (users may not realize what they're deleting)
- Revenue (users cancelling may prefer to cancel at period end rather than immediately)

### What Affects This
- Stripe billing portal configuration (must enable cancellation in portal settings)
- Grace period logic (PRD-084)
- Queued email cancellation (PRD-086)

### Related PRDs
- PRD-084 (Grace Period) -- grace period after cancellation needs clear UI communication

- PRD-086 (Emails After Deletion) -- confirmation should mention queued emails being cancelled
- PRD-083 (Double Checkout) -- similar UX concern for destructive actions

## Implementation Notes
- Use the existing `confirm-dialog.tsx` component (or create one based on shadcn/ui `AlertDialog`).
- The confirmation dialog should be a multi-step or checklist-style dialog:
  ```
  Are you sure you want to delete this organization?

  This will:
  - Cancel your Stripe subscription immediately
  - Disable all team members
  - Remove all domains
  - Cancel all pending emails

  You will have 7 days to re-subscribe before losing access.

  Type "DELETE" to confirm.
  ```
- For the "Cancel Subscription" button in billing tab, simply redirect to the Stripe billing portal where Stripe handles the cancellation UX natively. This is the simplest and most user-friendly approach.
- Backend change is optional for MVP: the Stripe portal handles cancellation at period end automatically.

## User Notes
- Users will see a clear, detailed confirmation dialog before deleting their org.
- A "Manage Subscription" button in the billing tab provides a Stripe-hosted way to cancel, pause, or update payment.
- The confirmation requires typing "DELETE" to prevent accidental deletion.
