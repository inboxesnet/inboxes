# PRD-061: Alias Lookup

## Status

P2-Medium

## Source

## Problem Statement
During inbound email processing, the fetch worker performs one database query per recipient address to look up the matching domain and alias. An email with 10 recipients in To/CC/BCC fields results in 10+ separate `SELECT FROM domains` queries and potentially 10+ `SELECT FROM aliases` queries. This per-recipient lookup pattern is the most impactful N+1 issue in the inbound email pipeline, as it runs on every incoming email.

## Current Behavior
In `backend/internal/queue/process_fetch.go:62-109`:
```go
for _, to := range emailData.To {
    parts := strings.Split(to, "@")
    // ...
    err := w.pool.QueryRow(ctx,
        "SELECT id FROM domains WHERE org_id = $1 AND domain = $2 AND status = 'active'",
        orgID, parts[1],
    ).Scan(&domainID)
    // ...
}
```

This pattern repeats for CC (lines 78-91) and BCC (lines 94-108), each with separate per-address queries.

Additionally, in the alias label stamping section (`process_fetch.go:348-359`):
```go
for _, addr := range append(append(emailData.To, emailData.CC...), emailData.BCC...) {
    // ...
    if tx.QueryRow(dbCtx,
        `SELECT address FROM aliases WHERE org_id=$1 AND address=$2`,
        orgID, cleanAddr,
    ).Scan(&matchedAlias) == nil {
        addLabelQ(dbCtx, tx, threadID, orgID, "alias:"+matchedAlias)
    }
}
```

This runs one query per recipient to check if it matches an alias.

The `routeEmail` function (`process_fetch.go:411-453`) also performs sequential queries per phase (user lookup, alias lookup, alias_users lookup), but this is less impactful since it only runs for the primary recipient.

## Expected Behavior
Batch the domain lookup and alias lookup into single queries:

1. **Domain lookup**: Extract all unique domains from To/CC/BCC addresses, then run one query:
   ```sql
   SELECT id, domain FROM domains WHERE org_id = $1 AND domain = ANY($2) AND status = 'active'
   ```

2. **Alias label stamping**: Collect all recipient addresses, then run one query:
   ```sql
   SELECT address FROM aliases WHERE org_id = $1 AND address = ANY($2)
   ```

## Files to Modify
- `backend/internal/queue/process_fetch.go:62-109` — Replace per-address domain lookups with batch query
- `backend/internal/queue/process_fetch.go:348-359` — Replace per-address alias lookups with batch query

## Implementation Notes
- For the domain lookup batch:
  ```go
  // Collect all unique domains from To/CC/BCC
  domainNames := make(map[string]bool)
  allRecipients := append(append(emailData.To, emailData.CC...), emailData.BCC...)
  for _, addr := range allRecipients {
      parts := strings.Split(addr, "@")
      if len(parts) == 2 {
          domainNames[parts[1]] = true
      }
  }
  uniqueDomains := make([]string, 0, len(domainNames))
  for d := range domainNames {
      uniqueDomains = append(uniqueDomains, d)
  }

  // Single batch query
  domainMap := make(map[string]string) // domain name -> domain ID
  rows, err := w.pool.Query(ctx,
      "SELECT id, domain FROM domains WHERE org_id = $1 AND domain = ANY($2) AND status = 'active'",
      orgID, uniqueDomains)
  // ... scan into domainMap ...
  ```

- Then iterate To/CC/BCC to find the first matching domain+recipient using the in-memory map instead of per-address queries.

- For the alias stamping batch:
  ```go
  cleanAddrs := make([]string, 0, len(allRecipients))
  for _, addr := range allRecipients {
      cleanAddrs = append(cleanAddrs, strings.ToLower(strings.TrimSpace(addr)))
  }
  aliasRows, err := tx.Query(dbCtx,
      "SELECT address FROM aliases WHERE org_id = $1 AND address = ANY($2)",
      orgID, cleanAddrs)
  // ... stamp labels for each matched alias ...
  ```

- The `aliases` table has an index on `(org_id)` but not on `(org_id, address)`. Consider adding a composite index as part of this change.

## User Notes
Inbound email processing will be faster, especially for emails with many recipients. Users will see no visible difference in behavior, but emails with large CC lists will appear in their inbox more quickly.
