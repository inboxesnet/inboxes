## Problem Statement
Several tabs in the Settings modal (Team, Aliases, Labels, Billing) show infinite loading spinners when their API calls fail. The `catch` blocks either swallow the error with an empty handler (`catch { // handled }`) or do not reset the loading state. The loading spinner never stops, and the user has no way to retry or even know that something went wrong.

## Current Behavior
In `frontend/components/settings-modal.tsx`:

- **Team tab** (`loadTeam`, line 492-501): `setTeamLoading(true)` is called at line 493, and `setTeamLoading(false)` is in the `finally` block at line 500. The `catch` block at line 497 has only a comment `// handled`. While the loading DOES reset (because of `finally`), no error message is shown to the user — the tab appears empty with no indication of failure.

- **Aliases tab** (`loadAliases`, line 589-604): `setAliasLoading(true)` at line 590, `setAliasLoading(false)` in `finally` at line 603. The `catch` at line 600 has only `// handled`. Same issue: loading resets but no error is displayed.

- **Labels tab** (`loadLabels`, around line 763-770): Same pattern: loading state resets but errors are swallowed.

- **Billing tab** (`loadBilling`, line 450-459): Same pattern: `catch { // handled }`.

- **Jobs panel** (`JobsPanel`, line 86-93): This one correctly shows an error state (`setError("Failed to load jobs")`) — it should be the model for the others.

The core issue is not infinite spinners (the `finally` blocks do work) but rather **silent failure**: tabs show empty content with no error message, no retry option, and no visual distinction between "no data exists" and "the API call failed."

## Expected Behavior
When an API call fails in any Settings modal tab:
1. The loading spinner should stop (already handled by `finally`).
2. An error message should be displayed: "Failed to load [team/aliases/labels/billing]. Click to retry."
3. A retry button or link should be available.
4. The error state should be visually distinct from the "empty" state (e.g., "No team members" vs. "Failed to load team").

## Implementation Notes
- Use the `JobsPanel` component (line 81-93 in settings-modal.tsx) as the reference implementation — it correctly handles loading, error, and empty states.
- Add per-tab error state variables: `teamError`, `aliasError`, `labelsError`, `billingError` (as strings).
- In each `catch` block, set the error state with a descriptive message.
- In each tab's render section, check for the error state before rendering the empty state. If error is set, show an error card with a retry button that calls the load function again.
- Consider a generic `TabErrorState` component: `<TabErrorState error={teamError} onRetry={loadTeam} />`.
- Clear the error state at the beginning of each load function (before `setLoading(true)`).

## User Notes
Currently, if the API is temporarily down and you open Settings > Team, you see an empty list with no explanation. After this fix, you'll see "Failed to load team members" with a "Retry" button.
