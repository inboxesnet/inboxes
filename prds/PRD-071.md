# PRD-071: Snooze & Send Later

## Features

**Snooze**: Hide a thread from inbox, resurface it at a chosen time. Thread gets a `snoozed_until` timestamp, filtered out of inbox view, reappears when the time hits.

**Send Later**: Compose an email, schedule it to send at a future time. Email saved as draft with a `scheduled_send_at` timestamp, background job picks it up and sends via Resend API when due.

## Shared Infrastructure

Both features need the same thing: a scheduler that checks for due items on a short interval (e.g. every 30-60 seconds) and acts on them.

- New background worker: `SchedulerWorker` polling for due snoozes and scheduled sends
- Or extend an existing worker (trash collector pattern — already polls on interval)

## Snooze

- Add `snoozed_until TIMESTAMPTZ` to `threads` table
- Frontend: snooze action on thread toolbar with time picker (1 hour, tomorrow morning, next week, custom)
- Thread filtered from inbox when `snoozed_until > now()`
- Worker unsnoozes by clearing `snoozed_until` + publishing WS event so thread reappears in real time
- Snoozed threads visible in a "Snoozed" view (like starred/trash)

## Send Later

- Add `scheduled_send_at TIMESTAMPTZ` to `drafts` table
- Frontend: dropdown on Send button → "Schedule send" with time picker
- Draft stays in drafts list with scheduled badge
- Worker picks up due drafts, sends via existing email send flow, converts draft to sent email
- If send fails, surface error to user (toast on next visit or email notification)
