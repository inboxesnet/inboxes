# PRD-107: Config Validation at Startup

## Status
Done

## Priority
P2-Medium

## Category
Configuration & Environment

## Source
GAP-FINDER C2

---

## Problem Statement
The `config.Load()` function in `backend/internal/config/config.go` performs minimal validation: it only checks that `SESSION_SECRET` and `ENCRYPTION_KEY` are non-empty, and validates Stripe cross-dependencies. Critical configuration values like `ENCRYPTION_KEY` format (should be base64-encoded 32 bytes), `SessionSecret` length, URL format for `AppURL`/`PublicURL`, port range for `APIPort`, and `DATABASE_URL` format are not validated. Invalid values cause cryptic runtime errors instead of clear startup failures.

## Current Behavior
In `config.go:30-74`, the `Load()` function:
- Lines 50-51: checks `SessionSecret != ""` but not minimum length
- Lines 53-54: checks `EncryptionKey != ""` but not that it's valid base64 or 32 bytes
- Line 37: accepts any string for `AppURL` (default `http://localhost:3000`)
- Line 38: accepts any string for `PublicURL` (default `http://localhost:8080`)
- Line 39: accepts any string for `APIPort` (default `8080`) -- e.g., `"abc"` would pass config loading but crash on `ListenAndServe`
- Line 32: accepts any string for `DatabaseURL` -- invalid URL causes a runtime pgx error
- Line 33: accepts any string for `RedisURL` -- invalid URL causes a runtime redis error
- Line 47: `EventRetentionDays` uses `getEnvInt` which silently falls back to default on parse error

## Expected Behavior
At startup, `config.Load()` should validate:
1. `ENCRYPTION_KEY`: must be valid base64, must decode to exactly 32 bytes
2. `SESSION_SECRET`: minimum 16 characters
3. `APP_URL` and `PUBLIC_URL`: must be valid URLs with http/https scheme
4. `API_PORT`: must be a valid port number (1-65535)
5. `DATABASE_URL`: must start with `postgres://` or `postgresql://`
6. `REDIS_URL`: must start with `redis://` or `rediss://`
7. `EVENT_RETENTION_DAYS`: must be positive integer (already handled by `getEnvInt`)

Invalid values should cause immediate, clear error messages and prevent startup.

## Files to Modify
- `backend/internal/config/config.go:30-74` -- Add validation checks after loading values
- `backend/internal/config/config_test.go` -- Add tests for validation (file already exists as untracked)

- Deployment reliability -- misconfigured instances fail fast with clear errors instead of cryptic runtime crashes

- Local development -- developers use defaults, validation should not block `localhost` defaults

## Implementation Notes
- Add validation after the config struct is populated but before returning:
  ```go
  // Validate ENCRYPTION_KEY format
  keyBytes, err := base64.StdEncoding.DecodeString(cfg.EncryptionKey)
  if err != nil {
      return nil, fmt.Errorf("ENCRYPTION_KEY must be valid base64: %w", err)
  }
  if len(keyBytes) != 32 {
      return nil, fmt.Errorf("ENCRYPTION_KEY must decode to 32 bytes, got %d", len(keyBytes))
  }

  // Validate SESSION_SECRET length
  if len(cfg.SessionSecret) < 16 {
      return nil, fmt.Errorf("SESSION_SECRET must be at least 16 characters")
  }

  // Validate URLs
  if _, err := url.Parse(cfg.AppURL); err != nil || !strings.HasPrefix(cfg.AppURL, "http") {
      return nil, fmt.Errorf("APP_URL must be a valid HTTP/HTTPS URL: %s", cfg.AppURL)
  }

  // Validate port
  port, err := strconv.Atoi(cfg.APIPort)
  if err != nil || port < 1 || port > 65535 {
      return nil, fmt.Errorf("API_PORT must be 1-65535, got: %s", cfg.APIPort)
  }
  ```
- Be careful with `ENCRYPTION_KEY` validation: the existing `service.NewEncryptionService()` already does this, so coordinate to avoid duplicate checks (or remove the check from there)
- Add a `WARN` log (not error) for `sslmode=disable` in DATABASE_URL when not running on localhost
- Test with `config_test.go` -- test both valid configs and each invalid case

## User Notes
This is a deployment reliability improvement. Catching configuration errors at startup saves hours of debugging in production.

## Acceptance Criteria
- [ ] `ENCRYPTION_KEY` that is not valid base64 causes a clear startup error
- [ ] `ENCRYPTION_KEY` that decodes to fewer or more than 32 bytes causes a clear startup error
- [ ] `SESSION_SECRET` shorter than 16 characters causes a clear startup error
- [ ] `API_PORT` with a non-numeric value causes a clear startup error
- [ ] `APP_URL` and `PUBLIC_URL` with invalid URL format cause clear startup errors
- [ ] Valid default values (localhost URLs, port 8080) pass validation for local development
- [ ] Error messages explicitly name the problematic variable and expected format
- [ ] Unit tests in `config_test.go` cover all validation cases
