# PRD-117: Dead Code — `/api/threads/unread-count` Superseded

## Status
Draft

## Priority
P3-Low

## Category
Dead Code

## Source
GAP-FINDER O2

---

## Problem Statement
The `/api/threads/unread-count` endpoint is registered and fully implemented but is superseded by `/api/domains/unread-counts`, which the frontend actually uses. The old endpoint remains as dead code, adding maintenance burden and an unnecessary query path against the database.

## Current Behavior
- `backend/internal/handler/threads.go:928-998` — The `UnreadCount` method is fully implemented with ~70 lines of code, including alias visibility filtering and both single-domain and all-domain query paths.
- `backend/internal/router/router.go:196` — The route is registered: `r.Get("/api/threads/unread-count", threads.UnreadCount)`.
- The frontend uses a different endpoint: `frontend/contexts/domain-context.tsx:38` calls `api.get<UnreadCounts>("/api/domains/unread-counts")`. A search of the entire `frontend/` directory for `threads/unread-count` returns zero results.
- The `UnreadCount` handler contains unchecked `.Scan()` calls on lines 966 and 995, contributing to the findings in PRD-049.

## Expected Behavior
The `/api/threads/unread-count` endpoint should be removed or deprecated. Since `/api/domains/unread-counts` provides the same data in a format the frontend expects, there is no reason to maintain two endpoints.

## Files to Modify
- `backend/internal/handler/threads.go:928-998` — Remove the `UnreadCount` method.
- `backend/internal/router/router.go:196` — Remove the route registration.

## Impact Analysis

### What This Affects
- No frontend functionality (endpoint is unused by the frontend).
- Removes ~70 lines of dead code from `threads.go`.
- Removes 2 unchecked `.Scan()` calls from the error handling audit scope.

### What Affects This
- If any external tooling or scripts call this endpoint, they would break. Since the app is pre-launch, this is unlikely.
- The `/api/domains/unread-counts` endpoint in `domains.go` serves the same purpose and must remain.

### Related PRDs
- PRD-049: Unchecked Queries — the `UnreadCount` handler has unchecked `.Scan()` calls on lines 966 and 995.
- PRD-116: Contacts Lookup — another dead endpoint that should be cleaned up.

## Implementation Notes
Before removing, confirm that no API documentation, Postman collections, or integration tests reference this endpoint. The `UnreadCount` method is the last method in `threads.go` (starting at line 928), so removal is straightforward.

The replacement endpoint `/api/domains/unread-counts` is in `backend/internal/handler/domains.go` and returns a map of `{domain_id: count}`, which is the format the frontend domain context expects.
