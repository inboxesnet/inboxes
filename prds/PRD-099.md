# PRD-099: Page Title Updates on Navigation

## Status
Done

## Priority
P3-Low

## Category
Accessibility

## Source
GAP-FINDER A5

---

## Problem Statement
The browser tab title stays fixed as "Inboxes -- The missing inbox for Resend" regardless of which folder or thread the user is viewing. Screen readers rely on the page title to announce the current view when the user switches tabs or navigates. Without dynamic titles, users cannot distinguish between Inbox, Sent, Drafts, or a specific thread without reading the page content.

## Current Behavior
The root layout in `frontend/app/layout.tsx:10-13` sets a static `metadata` object with `title: "Inboxes -- The missing inbox for Resend"`. Individual page components (e.g., `inbox/page.tsx`, `sent/page.tsx`, `drafts/page.tsx`) do not override the title. Thread view pages also do not update the title with the thread subject. The title never changes during client-side navigation.

## Expected Behavior
- Folder pages should update the title to: "Inbox - Inboxes", "Sent - Inboxes", "Drafts - Inboxes", etc.
- Thread view pages should update the title to: "{Subject} - Inboxes"
- The title should include unread count when applicable: "Inbox (3) - Inboxes"
- Screen readers should announce the new title when the user navigates between pages

## Impact Analysis

### What This Affects
- Screen reader page announcements on navigation
- Browser tab identification when multiple tabs are open
- WCAG 2.1 Level A compliance (Success Criterion 2.4.2: Page Titled)

### What Affects This
- Next.js App Router client-side navigation -- `metadata` export does not work in client components, so `document.title` must be set in `useEffect`
- Unread counts from `useDomains()` context for the "(3)" badge in the title
- Thread data loading -- title should update after the thread subject is available

### Related PRDs
- PRD-094 (ARIA Labels) -- general accessibility improvements
- PRD-103 (Toasts) -- notification announcements

## Implementation Notes
- Since all page components are `"use client"`, use `useEffect` to set `document.title`:
  ```tsx
  useEffect(() => { document.title = "Inbox - Inboxes"; }, []);
  ```
- For thread pages, update title when thread data loads:
  ```tsx
  useEffect(() => {
    if (thread?.subject) document.title = `${thread.subject} - Inboxes`;
  }, [thread?.subject]);
  ```
- Consider creating a `usePageTitle(title: string)` custom hook to standardize the pattern
- Reset title on unmount if needed, though Next.js navigation will trigger the new page's effect
- Include unread count for inbox: `Inbox (${unreadCount}) - Inboxes`
