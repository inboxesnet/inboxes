# PRD-102: aria-expanded on Collapsible Email Messages

## Status
Done

## Priority
P3-Low

## Category
Accessibility

## Source
GAP-FINDER A8

---

## Problem Statement
In the thread view, email messages can be expanded or collapsed by clicking on the sender row. However, there is no `aria-expanded` attribute on the clickable element, so screen readers cannot communicate whether a message is currently expanded or collapsed. Users relying on assistive technology have no way to know the state of each email message without reading its content.

## Current Behavior
In `thread-view.tsx`, the `EmailMessage` component (line 357-506) manages expansion state via `const [expanded, setExpanded] = useState(defaultExpanded)` at line 366. The collapsed view (line 384-402) is a `<button>` that triggers `setExpanded(true)` but has no `aria-expanded` attribute. The expanded view (line 408-436) has a `<button>` that triggers `setExpanded(false)` also without `aria-expanded`. Neither button has `aria-controls` pointing to the email content.

## Expected Behavior
- The toggle buttons for expanding/collapsing email messages should have `aria-expanded="true"` when expanded and `aria-expanded="false"` when collapsed
- The expandable content area should have an `id` that is referenced by `aria-controls` on the button
- Screen readers should announce "collapsed" or "expanded" when focusing the toggle

## Files to Modify
- `frontend/components/thread-view.tsx:385-388` -- Add `aria-expanded={false}` and `aria-controls={`email-content-${email.id}`}` to collapsed state button
- `frontend/components/thread-view.tsx:408-411` -- Add `aria-expanded={true}` and `aria-controls={`email-content-${email.id}`}` to expanded state button
- `frontend/components/thread-view.tsx:438` -- Add `id={`email-content-${email.id}`}` to the content `<div>` below the button
- `frontend/components/thread-view.tsx:384-402` -- Add `aria-label` to collapsed button: "Expand email from {sender}"
- `frontend/components/thread-view.tsx:408-436` -- Add `aria-label` to expanded button: "Collapse email from {sender}"

## Implementation Notes
- The same button element already handles both states; just add `aria-expanded={expanded}` to both the collapsed and expanded button elements
- Use the email `id` to generate unique `aria-controls` IDs: `email-content-${email.id}`
- Example:
  ```tsx
  <button
    onClick={() => setExpanded(!expanded)}
    aria-expanded={expanded}
    aria-controls={`email-content-${email.id}`}
    aria-label={expanded ? `Collapse email from ${formatSenderName(email.from_address)}` : `Expand email from ${formatSenderName(email.from_address)}`}
  >
  ```
- The content div should receive the matching `id`:
  ```tsx
  <div id={`email-content-${email.id}`} className="px-4 pb-4">
  ```
