# PRD-002: Add Composite Index on aliases(org_id, address)

## Problem

`aliases` table has separate indexes on `org_id` and `address` but no composite index. Queries filtering by both (e.g. alias lookup during email processing) can't use a single efficient index scan.

## Fix

One migration:

```sql
CREATE INDEX CONCURRENTLY idx_aliases_org_address ON aliases (org_id, address);
```

Use `CONCURRENTLY` to avoid locking the table during creation.

## Priority

Low. No impact until alias volume is high, but trivial to add.
