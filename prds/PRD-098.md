# PRD-098: Skip Navigation Link

## Status
Done

## Priority
P3-Low

## Category
Accessibility

## Source

## Problem Statement
There is no "Skip to main content" link in the app layout. Keyboard users must tab through the entire sidebar (domain icons, compose button, all folder labels, custom labels, theme toggle, settings, logout) before reaching the main content area. On the desktop layout, this means tabbing through 15-20+ interactive elements in the sidebar before reaching the thread list or thread view.

## Current Behavior
The domain layout in `frontend/app/(app)/d/[domainId]/layout.tsx` renders the sidebar (`DomainSidebar`) before the `<main>` element (line 151). The sidebar contains domain icon buttons (one per domain), a compose button, 7 folder labels, custom labels, and 3 utility buttons (theme, settings, logout). There is no skip link anywhere in the component tree. The `<main>` element at line 151 has no `id` for skip link targeting.

## Expected Behavior
- A visually hidden "Skip to main content" link should appear as the first focusable element in the page
- When focused (via Tab), it should become visible (standard skip link pattern)
- Activating the link should move focus to the `<main>` content area
- This allows keyboard users to bypass the sidebar with a single keypress

## Files to Modify
- `frontend/app/(app)/d/[domainId]/layout.tsx:151` -- Add `id="main-content"` and `tabIndex={-1}` to the `<main>` element
- `frontend/app/(app)/d/[domainId]/layout.tsx:128` -- Add skip link as first child inside the layout container div
- `frontend/app/globals.css` -- Add CSS for the visually-hidden-but-focusable skip link pattern

## Implementation Notes
- Standard skip link CSS pattern:
  ```css
  .skip-link {
    position: absolute;
    left: -9999px;
    z-index: 999;
  }
  .skip-link:focus {
    left: 50%;
    transform: translateX(-50%);
    top: 4px;
    padding: 8px 16px;
    background: var(--background);
    border: 1px solid var(--border);
    border-radius: 4px;
  }
  ```
- The skip link should be an `<a href="#main-content">Skip to main content</a>`
- Consider also adding `role="main"` to the `<main>` element (it's implicit but makes it explicit)
- Optionally add a second skip link "Skip to compose" for quick email composition access

## User Notes
