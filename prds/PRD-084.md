# PRD-084: Grace Period

P1-High

## Problem Statement
When a subscription is cancelled, the org receives a 7-day grace period (`plan_expires_at` set to `now() + 7 days`). The `RequirePlan` middleware correctly checks this expiry at request time. However, after the grace period expires, no background job transitions the org from `plan: "cancelled"` with an expired `plan_expires_at` to `plan: "free"`. The org remains in a limbo state where `RequirePlan` correctly blocks access, but the plan column never reflects the true state ("free"), causing confusion in the billing UI and admin tooling.

## Current Behavior
In `backend/internal/handler/billing.go:254-276`, when `customer.subscription.deleted` is received:
```go
tag, err := h.DB.Exec(ctx,
    `UPDATE orgs SET plan = 'cancelled', plan_expires_at = $1, stripe_subscription_id = NULL
     WHERE stripe_customer_id = $2`,
    time.Now().Add(7*24*time.Hour), sub.Customer.ID,
)
```

The `RequirePlan` middleware in `backend/internal/middleware/auth.go:176-179` checks:
```go
if plan == "cancelled" && planExpiresAt != nil && planExpiresAt.After(time.Now()) {
    next.ServeHTTP(w, r)
    return
}
```

After the grace period expires, `planExpiresAt.After(time.Now())` returns false, so access is blocked. But the plan column still says `"cancelled"`, never transitioning to `"free"`. The billing UI in `settings-modal.tsx:1136` shows "Cancelled" indefinitely instead of "Free".

## Expected Behavior
1. A background worker should run periodically (e.g., every hour) to find orgs where `plan = 'cancelled' AND plan_expires_at < now()`.
2. For these orgs, set `plan = 'free'` and `plan_expires_at = NULL`.
3. Emit a `plan.changed` WebSocket event so connected clients update their UI.
4. The billing UI should show "Free" for orgs whose grace period has expired.

## Files to Modify
- `backend/internal/worker/` -- New file: `grace_period.go` with a periodic worker that transitions expired cancelled plans to free
- `backend/cmd/api/main.go` -- Start the grace period worker alongside other workers

- `frontend/components/settings-modal.tsx:1136` -- Ensure UI handles `plan: "free"` correctly after grace period

## Implementation Notes
- Model after `EventPruner` in `backend/internal/worker/event_pruner.go` -- periodic ticker with a simple UPDATE query.
- Query: `UPDATE orgs SET plan = 'free', plan_expires_at = NULL, updated_at = now() WHERE plan = 'cancelled' AND plan_expires_at < now() RETURNING id, stripe_customer_id`
- For each returned org, publish a `plan.changed` event with `plan: "free"`.
- Run interval: every 1 hour (grace period is 7 days, so hourly precision is more than sufficient).
- Add the interval as a configurable env var: `GRACE_PERIOD_CHECK_INTERVAL`.

## User Notes
- After their grace period expires, users will see "Free Plan" in billing settings instead of "Cancelled".
- No change to access behavior -- access was already correctly blocked by `RequirePlan`.
