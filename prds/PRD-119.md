# PRD-119: Dead Code — `truncate()` Utility Function Never Imported

## Problem Statement
The `truncate()` function is defined in `frontend/lib/utils.ts` but is never imported or used by any component or other module. It adds dead code that gets included in the bundle and creates false expectations about its usage during code review.

## Current Behavior
- `frontend/lib/utils.ts:85-88` — The function is defined:
  ```typescript
  export function truncate(str: string, maxLen: number): string {
    if (str.length <= maxLen) return str;
    return str.slice(0, maxLen - 1) + "\u2026";
  }
  ```
- A search for `import.*truncate` across the entire `frontend/` directory returns zero results — no component imports this function.
- `frontend/lib/__tests__/utils.test.ts:145-155` — There are tests for this function (lines 145-155), which import it directly from `utils.ts`.
- The function is tree-shakeable in production builds, but its presence (along with its tests) adds maintenance overhead.

## Expected Behavior
Two options:
1. **Remove** the function and its tests, since no code uses it. If needed later, it can be recreated.
2. **Keep** it if there is a planned use case (e.g., truncating thread subjects in the thread list, truncating long email addresses). In this case, add a comment documenting the intended future use.

## Files to Modify
- `frontend/lib/utils.ts:85-88` — Remove the `truncate` function (or add usage comment if keeping).
- `frontend/lib/__tests__/utils.test.ts:145-155` — Remove the `truncate` test block (or keep if keeping the function).

If removing, also update the test file import to remove `truncate` from the destructured import on line 7.
