# PRD-104: Keyboard Alternative for Drag-and-Drop

## Status
Done

## Priority
P3-Low

## Category
Accessibility

## Source
GAP-FINDER A10

---

## Problem Statement
Moving threads between folders (inbox, archive, trash, spam, custom labels) is only possible via mouse drag-and-drop using `@dnd-kit/core`. There is no keyboard-accessible alternative for this functionality. Users who cannot use a mouse (motor disabilities, screen reader users) cannot move threads between folders using drag-and-drop. While bulk actions in the toolbar provide some of this functionality, single-thread folder moves from the thread list rely entirely on drag.

## Current Behavior
In `frontend/app/(app)/d/[domainId]/layout.tsx:122-174`, a `DndContext` wraps the entire layout. `ThreadRow` in `thread-list.tsx:60-63` uses `useDraggable()` from `@dnd-kit/core`. `DroppableLabelButton` in `domain-sidebar.tsx:54-92` uses `useDroppable()`. The `DragPreview` component (`drag-preview.tsx:1-25`) shows a visual preview during drag. The `DndContext` has `accessibility={{ screenReaderInstructions: { draggable: "" } }}` at line 126, which disables the default screen reader instructions (empty string).

The only keyboard-accessible way to move threads is: select threads (checkbox or `x` key), then use toolbar bulk actions. But there is no "Move to..." option in the toolbar that presents all available destinations. The toolbar only shows context-specific actions (Archive if in inbox, Move to Inbox if in archive/trash).

## Expected Behavior
- A keyboard-accessible "Move to..." command should be available for selected threads
- Options: keyboard shortcut (e.g., `v` for "move to" like Gmail) that opens a folder picker
- Or: a "Move to" dropdown button in the toolbar/thread view with all available destinations
- The drag-and-drop should have meaningful screen reader instructions (not empty string)

## Files to Modify
- `frontend/app/(app)/d/[domainId]/layout.tsx:126` -- Provide meaningful screen reader instructions for drag-and-drop instead of empty string
- `frontend/components/thread-toolbar.tsx` -- Add a "Move to..." dropdown button in the bulk actions area
- `frontend/components/keyboard-shortcuts.tsx` -- Add `v` key shortcut to open "Move to" dialog
- `frontend/components/keyboard-shortcuts-dialog.tsx:12-23` -- Document the new shortcut
- `frontend/components/thread-view.tsx:237-258` -- Optionally add "Move to..." button in thread view toolbar

## Impact Analysis

### What This Affects
- Keyboard-only users' ability to organize emails into folders
- Screen reader users' understanding of drag-and-drop functionality
- WCAG 2.1 Level A compliance (Success Criterion 2.1.1: Keyboard)

### What Affects This
- The `@dnd-kit/core` library's built-in keyboard sensor -- it exists but requires configuration
- Custom labels from the API (`/api/labels`) need to be included in the move destinations
- Thread action mutation (`useThreadAction`) already supports `move:inbox`, `move:archive`, etc.

## Implementation Notes
- Option A (simplest): Add a "Move to" dropdown in the toolbar that lists all folders + custom labels. On click, calls the existing `onBulkAction("move:{label}")` handler.
- Option B: Implement `v` keyboard shortcut (Gmail-style) that opens a popover/dialog with folder list
- Option C: Configure `@dnd-kit` KeyboardSensor with proper announcements:
  ```tsx
  import { KeyboardSensor } from "@dnd-kit/core";
  const keyboardSensor = useSensor(KeyboardSensor);
  ```
- The existing bulk action handler already supports `move:inbox`, `archive`, `trash`, `spam`, and `label:{name}` -- the infrastructure is there
- For screen reader instructions, replace the empty string with:
  ```tsx
  accessibility={{
    screenReaderInstructions: {
      draggable: "To pick up a thread, press Space or Enter. Use arrow keys to move. Press Space or Enter to drop, or Escape to cancel."
    }
  }}
  ```

## User Notes
