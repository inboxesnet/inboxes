# PRD-042: Rate Limit Range

## Status
Done

## Priority
P2-Medium

## Category

## Source

## Problem Statement
The Resend API rate limit setting in the organization settings panel has incomplete frontend validation. The backend enforces a range of 1-100 at `orgs.go:73` but the frontend Input at `settings-modal.tsx:1993-1996` only uses `min={1}` and `max={100}` HTML attributes (which are advisory for `type="number"` and can be bypassed). Entering non-numeric values like "abc" silently defaults to `2` via `parseInt(e.target.value) || 2`, and there is no visible indication of the valid range.

## Current Behavior
- In `frontend/components/settings-modal.tsx:1993-1996`, the rate limit input uses `type="number"`, `min={1}`, `max={100}`, with `onChange` handler `parseInt(e.target.value) || 2`.
- Typing "abc" results in `NaN`, which the `|| 2` fallback converts to `2`. The user sees "2" replace their input with no explanation.
- Typing "150" is accepted by the input (HTML min/max on number inputs do not prevent typing, only affect form validation). On save, the backend at `orgs.go:73` returns: "resend_rps must be between 1 and 100".
- Typing "0" or "-5" is similarly accepted by the input and rejected by the backend.
- There is a help text at line 2001-2003 about the default being 2 and contacting Resend for higher limits, but no mention of the 1-100 range.

## Expected Behavior
- The frontend should validate the rate limit value is a number between 1 and 100 before submitting.
- The input should show a clear range indicator (e.g., "1-100 requests/second").
- Invalid values should be caught with an inline error message before the API call.

## Files to Modify
- `frontend/components/settings-modal.tsx:1989-2003` -- Add range validation and update help text to show valid range
- `frontend/components/settings-modal.tsx:~850` -- Add client-side validation in the org settings save handler before API call

## Implementation Notes
- Update the save handler to validate before API call:
  ```typescript
  if (orgResendRPS < 1 || orgResendRPS > 100 || isNaN(orgResendRPS)) {
    toast.error("Rate limit must be between 1 and 100");
    return;
  }
  ```
- Update help text from "Default is 2 (Resend's standard limit)" to "Must be between 1 and 100. Default is 2 (Resend's standard limit)."
- Consider clamping the value on blur rather than on every keystroke to avoid interfering with typing.
- The `onChange` handler should handle the empty string case (user deletes content) without defaulting to 2 silently.

- Invalid values will be caught before saving, with a clear error message.
