# PRD-072: Forwarding Rules & Auto-Reply

## Features

**Forwarding Rules**: Automatically forward incoming emails matching certain criteria to another address. Think Gmail filters → "Forward to X".

**Auto-Reply**: Automatically reply to incoming emails with a canned response. Think vacation responder / out-of-office.

## Forwarding Rules

- New `forwarding_rules` table: `id, org_id, domain_id, alias_id (nullable), conditions (jsonb), forward_to, enabled, created_at`
- Conditions: from address, to address, subject contains, etc. (start simple — exact match on from/to, expand later)
- Webhook handler checks rules after inserting email, fires off forward via Resend API if matched
- Forwarded emails tracked (avoid loops — don't forward forwarded emails)

## Auto-Reply

- New `auto_replies` table: `id, org_id, domain_id, alias_id (nullable), subject, body_html, enabled, start_at, end_at (nullable), created_at`
- Optional time window (vacation: start/end date) or always-on
- Webhook handler checks after inserting email, sends reply via Resend API if matched
- Rate limit: one auto-reply per sender per 24h (prevent reply loops)
- Don't auto-reply to noreply@, mailer-daemon@, or other auto-generated addresses

## Shared Infrastructure

Both hook into the webhook email processing pipeline. After an email is received and stored, check forwarding rules then auto-reply rules.

## Priority

Not built. Future feature — no timeline. Manual forwarding (compose + forward) already exists.
