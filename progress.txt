# Progress Log

## Learnings
- create-next-app interactive prompts don't work well in non-interactive shells; manually create files instead
- shadcn/ui needs: class-variance-authority, clsx, tailwind-merge, lucide-react, tailwindcss-animate
- shadcn/ui components.json + lib/utils.ts with cn() helper is the foundation
- tailwind.config.ts needs CSS variable-based colors and tailwindcss-animate plugin
- globals.css needs @layer base with CSS variables for light/dark themes
- Project name must be lowercase in package.json for npm compatibility
- Prisma 7 removes `url` from datasource block; downgraded to Prisma 5 for traditional config
- Use `DATABASE_URL="..." npx prisma validate` to validate schema without .env file
- Use `prisma migrate diff --from-empty --to-schema-datamodel` to generate migration SQL without DB connection
- Prisma enums use lowercase values: `admin`, `member`, `invited`, etc.
- migration_lock.toml must exist in prisma/migrations/ with `provider = "postgresql"`
- For incremental migrations without a DB, write SQL manually based on schema diff
- Use `bcryptjs` (pure JS) for bcrypt hashing; `jose` for JWT operations
- Prisma client singleton at lib/db.ts; session utils at lib/session.ts
- `npx prisma generate` must run before typecheck for @prisma/client types
- Next.js 14 App Router: cookies() is async, API routes use NextRequest/NextResponse

- What was implemented: POST /api/domains endpoint — admin-only, validates domain format (regex for valid TLD, no protocol/path), creates Domain record with status=pending, calls Resend API to register domain, returns DNS records (type, name, value, priority), returns 409 if domain already registered
- Files changed: app/api/domains/route.ts (new)
- Learnings for future iterations:
  - Resend domain registration: POST https://api.resend.com/domains with { name: domain }
  - Resend returns { id, records: [...] } where each record has type, name, value, priority, status
  - Domain validation regex: /^(?!-)([a-zA-Z0-9-]{1,63}(?<!-)\.)+[a-zA-Z]{2,}$/ — validates proper TLD format
  - Admin-only endpoints: check user.role !== "admin" and return 403
  - getCurrentUser() already returns null for unauthenticated (handled by middleware), but double-check in route
  - Pattern: validate input → check conflicts (409) → call external API → save to DB → return response

- Files changed: app/api/domains/[id]/verify/route.ts (new)
- Learnings for future iterations:

  - DKIM records identified by CNAME/TXT type with name containing "dkim"
  - Use `findFirst` with org_id filter to ensure domain belongs to user's org (security)
  - DomainStatus enum values: "pending", "verified", "active" — cast string with `as` for type safety
